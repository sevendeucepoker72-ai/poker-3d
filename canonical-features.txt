# Canonical Features Manifest — americanpubpoker.online (poker-3d)
#
# PURPOSE: prevent feature regressions on deploy. The FTP deploy script
# (deploy-online.mjs) greps the BUILT bundle in dist/assets/** for every
# non-comment/non-blank line in this file and ABORTS the deploy if any
# are missing.
#
# Earned 2026-05-06 after a regression where the standalone poker-3d
# tree at Downloads/developer setup/poker-3d/ had Phase 2 #5 native-OIDC
# work that the mono-repo mirror at apps/poker-3d/ was missing. The FTP
# deploy ran from the mono-repo mirror and silently shipped the stale
# pre-OIDC build — Play Online sign-in broke. This manifest exists so
# the next tree-drift is caught at deploy time instead of in production.
#
# RULES
# - One token per line. Tokens matched as plain substrings.
# - **Tokens MUST be minification-safe**: string literals (URL strings,
#   localStorage keys, event names, error class names, JSON values, log
#   tags). Function/variable/import names get mangled by Vite/Terser to
#   single letters and will NOT survive — don't lock those.
# - Lines starting with `#` are comments. Blank lines ignored.
# - To deliberately retire a feature: DELETE the line in the SAME commit
#   that retires the feature. The diff is the audit trail.

# ─── 2026-05-07 OAuth audit P0 fixes ─────────────────────────────────
# iOS PWA query-string stripping fix in authService.getCallbackParams.
# On iOS PWA cold launches, window.location.search arrives empty; the

# makes login work for Home Screen launches. The cache key is a unique

# Cross-tab refresh single-flight lock (ported from player-web). The

# ─── 2026-05-07 — cross-site SSO via bridge tokens ───────────────────
# Cookie-based cross-site SSO is dead in modern Chrome (CHIPS partition
# keys, 3p cookie blocking). Replaced with bridge tokens: each outbound
# link to a sibling site appends #bridge_id_token=<jwt>. Receiving site
# exchanges via grant_type=urn:apk:bridge for its own full token set.
# Anchor on the unique grant type name + the URL fragment key.
urn:apk:bridge
bridge_id_token

# ─── 2026-05-12 — CLAUDE.md Pattern B self-heal (port from player-web) ─
# Roles / VIP fields are refreshed from /users/:id/me on mount AND on
# every tab-resume, so an admin grant/revoke or VIP extension reaches
# this tab without sign-out + sign-in. Without this, a field stamped
# once at handleLogin stays stuck on its initial value forever — exactly
# the Pattern B regression that broke dealer hubs on player-web in May
# 2026. Both log tags are emitted ONLY when the value actually changes.
[dealer-auto-heal] userRoles merged from /me
[vip-auto-heal] isVip merged from /me
